# FitLog

**v1.2 要件仕様書**

テスト・品質強化

2026年4月 / Version 1.2

---

## 1. 概要

v1.1 で構築した認証・ユーザー別データ分離の基盤に、テストと CI を追加する。
Vitest によるユニットテスト、Playwright による E2E テスト、GitHub Actions による自動実行を整備し、継続的な品質担保の仕組みを作る。

---

## 2. 追加する技術スタック

| カテゴリ | 技術 | 用途 |
|---|---|---|
| ユニットテスト | Vitest | スキーマ・ユーティリティ関数の単体テスト |
| E2E テスト | Playwright | ブラウザ操作を通じたユーザーフローのテスト |
| CI | GitHub Actions | PR・push 時のテスト自動実行 |

---

## 3. テスト対象と方針

### 3.1 ユニットテスト（Vitest）

Server Actions / API クライアントは DB 接続が必要なためユニットテスト対象外とする。
純粋関数・スキーマなど、DB や外部依存のないロジックをテストする。

| 対象ファイル | テスト内容 |
|---|---|
| `schemas/bodyRecord.ts` | バリデーション（正常値・境界値・異常値） |
| `schemas/goal.ts` | バリデーション（正常値・境界値・異常値） |
| `utils/date.ts` | 日付フォーマット関数 |

### 3.2 E2E テスト（Playwright）

実際のブラウザでユーザーフローを検証する。
認証フローは Google OAuth のため E2E テストから除外し、認証済みセッションを前提とする。

| テストケース | 内容 |
|---|---|
| 体重記録の登録 | フォームから記録を入力して保存できる |
| 体重記録の編集 | 既存レコードを編集して上書きできる |
| 体重記録の削除 | レコードを削除できる |
| 目標の設定 | 目標体重・体脂肪率を設定できる |

### 3.3 テストしないもの

| 対象 | 理由 |
|---|---|
| Google OAuth フロー | 外部サービスのため E2E テスト困難 |
| Server Actions / API クライアント | DB 接続が必要。統合テストは将来のバージョンで検討 |
| UI コンポーネントの見た目 | スナップショットテストは変更コストが高いため導入しない |

## 5. CI 仕様（GitHub Actions）

### トリガー

- `main` ブランチへの push
- すべてのブランチへの PR 作成・更新

### ジョブ構成

```
ci
├── lint        ESLint による静的解析
├── typecheck   TypeScript 型チェック（tsc --noEmit）
├── unit-test   Vitest ユニットテスト
└── e2e         Playwright E2E テスト（unit-test 成功後に実行）
```

### E2E の実行環境

- Playwright の `storageState` を使い、テスト用アカウントで認証済みセッションを再現する
- 接続先 DB はローカルの `.env` ではなく GitHub Actions の Secrets を参照する
