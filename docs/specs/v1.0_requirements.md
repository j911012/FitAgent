# FitLog

**v1.0 要件仕様書**

体重・体脂肪率記録・可視化 Webアプリ

2026年4月 / Version 1.0

---

## 1. 概要

体重・体脂肪率を記録・可視化するWebアプリ。シンプルなCRUDから始め、バージョンアップで機能追加していく。最終的にはTDEE/基礎代謝ベースのカロリー管理、RPE・漸進性過負荷に基づくトレーニング管理、睡眠・疲労管理を統合したエージェント搭載の総合健康管理アプリを目指す。

### 1.1 プロジェクト目的

- Next.js / TypeScriptを用いたモダンWeb開発の学習
- 設計・バックエンド・ネットワーク・API等のエンジニアスキルの網羅的習得
- v1.0は最小限の機能でリリースし、段階的に拡張

## 5. 機能仕様

### 5.1 記録（Create）

- 「体重を記録」ボタン押下でモーダル表示
- 入力項目: 体重（必須・数値・小数点1桁）、体脂肪率（任意・数値・小数点1桁）
- 同日に既存レコードがある場合はUPSERT（上書き）
- 保存後: モーダルを閉じ、画面をリフレッシュ

### 5.2 編集（Update）

- 一覧のペンアイコン押下で記録モーダルを表示（既存値をフィルイン）
- 保存後: モーダルを閉じ、画面をリフレッシュ

### 5.3 削除（Delete）

- 一覧のゴミ箱アイコン押下
- 確認なしで即削除（v1.0はシンプルに）
- 削除後: 画面をリフレッシュ

### 5.4 目標設定

- 設定ボタン（歯車アイコン）押下で目標設定モーダルを表示
- 入力項目: 目標体重（任意・数値・小数点1桁）、目標体脂肪率（任意・数値・小数点1桁）
- 保存後: モーダルを閉じ、プログレスバーに即反映
- 目標未設定時: プログレスバーは「目標未設定」とグレー表示
- 目標達成時: プログレスバーが緑になり「達成！」と表示

## 6. バリデーション仕様

| フィールド          | ルール                                     |
| ------------------- | ------------------------------------------ |
| 体重 (weight_kg)    | 必須。数値。範囲: 20.0〜300.0kg。小数点1桁 |
| 体脂肪率 (body_fat) | 任意。数値。範囲: 3.0〜60.0%。小数点1桁    |
| 目標体重            | 任意。数値。範囲: 20.0〜300.0kg。小数点1桁 |
| 目標体脂肪率        | 任意。数値。範囲: 3.0〜60.0%。小数点1桁    |
| 日付 (date)         | 自動付与（今日）。未来日は不可             |

バリデーションはServer Actions側（zod）で実施。クライアント側は `type="number"` のみ。

## 11. ロードマップ

以下のバージョンアップを段階的に実施し、最終的にTDEE/基礎代謝ベースのカロリー管理、RPE・漸進性過負荷に基づくトレーニング管理、睡眠・疲労管理を統合したエージェント搭載の総合健康管理アプリを目指す。

---

### Phase 1 — 基盤整備

#### v1.0 — MVP（現在）

- 体重・体脂肪率のCRUD
- SVGグラフ（直近30日）
- 目標設定・プログレスバー

#### v1.1 — 認証・ユーザー分離

- Auth.js（NextAuth）導入
- ユーザー別データ分離（`user_id` カラム追加）
- Neon Row Level Security（RLS）設定
- `.env` 周りのセキュリティ強化

#### v1.2 — テスト・品質強化

- Vitest導入、ユニットテスト整備
- E2EテストにPlaywright導入
- GitHub Actions CI強化（テスト自動実行）

#### v1.3 — API Route化

- Server ActionsをREST API（Route Handlers）に移行
- OpenAPI（Swagger）ドキュメント整備
- APIバリデーション強化（zodスキーマ共通化）

---

### Phase 2 — トレーニング記録（RPE / 漸進性過負荷）

#### v2.0 — トレーニング基盤

- `exercises` マスタテーブル（種目名・部位・カテゴリ: コンパウンド/アイソレーション）
- `workout_sessions` テーブル（日付・セッション名・メモ）
- `workout_sets` テーブル（種目・セット番号・重量・レップ数）
- トレーニング記録CRUD（セッション単位での登録・編集・削除）
- トレーニング履歴一覧・種目別フィルタ

#### v2.1 — RPE・ボリューム管理

- `workout_sets` にRPE（主観的運動強度）カラム追加（1〜10スケール）
- RPEから1RM推定値の自動計算（Epley式 / RPEベース）
- セットボリューム（重量 × レップ数 × セット数）の自動集計
- 種目別ボリューム推移グラフ
- RPE分布グラフ（オーバートレーニング検知の基礎）

#### v2.2 — 漸進性過負荷トラッキング

- 種目別の重量・レップ数・ボリュームの週次推移グラフ
- 前回セッションとの自動比較（重量・レップ数の増減表示）
- プログレッシブオーバーロードアラート（停滞検知）
- 1RM推定値の推移グラフ（筋力の伸びを可視化）
- 部位別ボリュームバランスチャート（胸・背中・脚等の偏り検知）

---

### Phase 3 — カロリー・栄養管理（TDEE / 基礎代謝）

#### v3.0 — プロフィール・基礎代謝算出

- `user_profiles` テーブル（身長・年齢・性別・活動レベル）
- BMR自動計算（Mifflin-St Jeor式）
- TDEE自動計算（BMR × 活動係数）
- 目標別カロリー目安表示（減量: TDEE-500kcal / 維持: TDEE / 増量: TDEE+300kcal）
- 体重変動からの実際のTDEE逆算（週次平均体重から算出）

#### v3.1 — 食事記録CRUD

- `meal_records` テーブル（日付・食事タイプ: 朝/昼/夜/間食）
- `meal_items` テーブル（食品名・カロリー・タンパク質・脂質・炭水化物・量）
- 食事記録CRUD（食事単位での登録・編集・削除）
- 1日の摂取カロリー / PFCバランス自動集計

#### v3.2 — カロリーダッシュボード

- 日次カロリーバランスグラフ（摂取 vs TDEE）
- PFCバランス円グラフ（目標比率との乖離表示）
- 週次平均カロリー推移グラフ
- 体重変動 × カロリー摂取の相関グラフ

#### v3.3 — 外部API連携

- 食品データベースAPI連携（カロリー・栄養素の自動取得）
- 食品検索・オートコンプリート入力
- よく使う食品のお気に入り登録

---

### Phase 4 — 睡眠・疲労・リカバリー管理

#### v4.0 — 睡眠記録

- `sleep_records` テーブル（就寝時刻・起床時刻・睡眠時間自動計算）
- 睡眠の質評価（5段階主観評価）
- 睡眠記録CRUD
- 直近30日の睡眠時間推移グラフ

#### v4.1 — 疲労度・コンディション管理

- 日次コンディション記録（筋肉痛・ストレスレベル・体調）
- 疲労度スコア自動算出（トレーニングボリューム + RPE + 睡眠時間 + 主観評価）
- 疲労トレンドグラフ（過去7日の移動平均）
- オーバートレーニング警告（疲労度スコアが閾値超過時）
- 休息日レコメンドロジック（疲労スコアベース）

#### v4.2 — リカバリーダッシュボード

- 睡眠 × トレーニング × 疲労の統合ダッシュボード
- パフォーマンススコア（睡眠質 + 疲労度 + トレーニングボリュームの総合指標）
- 「今日のコンディション」サマリーカード
- 週次レポート（睡眠・疲労・トレーニングのサマリー）

---

### Phase 5 — 統合ダッシュボード・データ分析

#### v5.0 — 統合ダッシュボード

- 体重・カロリー・トレーニング・睡眠のクロスドメインダッシュボード
- 複数指標の相関分析グラフ（体重 × カロリー × トレーニング量 × 睡眠）
- 週次・月次サマリーカード
- データエクスポート（CSV / JSON）

#### v5.1 — 目標設定高度化

- 期間指定の目標設定（〇月〇日までに△kg）
- 達成ペース予測グラフ
- 目標達成バッジ・ストリーク管理

#### v5.2 — 通知・リマインダー

- 記録忘れプッシュ通知（Web Push API）
- 週次レポートメール送信
- 体型写真アップロード（Vercel Blob）・時系列比較ビュー

---

### Phase 6 — AIエージェント統合

#### v6.0 — AIアドバイス機能

- 記録データをもとにした健康アドバイス生成
- 食事改善提案（PFCバランス最適化・目標カロリー達成のためのメニュー提案）

#### v6.1 — トレーニングAI

- 漸進性過負荷の原則に基づく次回トレーニングメニュー自動生成
- RPE・ボリューム・疲労度を考慮した負荷調整提案
- 停滞期検知時のディロード・ピリオダイゼーション提案
- 部位別ボリューム偏りの自動検知と改善提案

#### v6.2 — リカバリーAI

- 疲労度・睡眠・トレーニング履歴を統合した休息日提案
- オーバートレーニング予防のためのディロードウィーク自動提案
- 睡眠改善アドバイス（睡眠時間・質のトレンドから）

#### v6.3 — エージェント化

- チャットUIでAIと対話しながら記録・相談
- 長期トレンド分析・異常検知アラート
- パーソナライズされた週次プラン自動生成（トレーニング + 食事 + 休息）
- 自然言語での記録入力（「今日ベンチ100kg3レップ」→ 自動パース）
